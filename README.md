# Medicaid Claims Fraud Detection Pipeline

A 24-step Python analysis pipeline for detecting fraud, waste, and abuse patterns in CMS Medicaid provider spending data. Analyzes **$1.09 trillion** in payments across **227 million** billing records, **617,503 providers**, and **10,881 procedure codes** spanning January 2018 through December 2024.

## Key Findings

- **$355 billion** in suspicious provider-level spending identified across 10 distinct fraud patterns
- **$116 billion** in system-wide rate and policy anomalies
- **500 high-risk providers** prioritized for investigation, 90% flagged by 3+ independent detection methods
- 11 of the top 20 flagged entities are government agencies — pointing to rate design issues, not just bad actors

## Pipeline Architecture

The pipeline runs 24 milestones sequentially via `scripts/run_all.py`:

```
00  CSV Data Quality Scan
01  Data Ingestion (CSV → DuckDB)
02  Reference Data Enrichment (NPPES, HCPCS, LEIE)
03  Exploratory Data Analysis
04  Hypothesis Generation
05  Statistical Hypothesis Testing (Categories 1-5)
06  ML/DL Anomaly Detection (Isolation Forest, Random Forest, XGBoost)
07  Domain-Specific Rules
08  Cross-Reference & Composite Scoring
09  Financial Impact & Deduplication
10  Chart Generation
11  Report Assembly (CMS Administrator Report)
12  Feasibility Matrix + Hypothesis Validation
13  Longitudinal Panel Build + Multivariate Analysis
14  Validation & Calibration
15  DQ Atlas State Quality Weights
16  Risk Queue Pack
17  Executive Dashboard Cards
18  Hypothesis Cards
19  Executive Brief
20  Merged Cards
21  Fraud Pattern Summary
22  Action Plan Memo
23  Provider Validation Scores
```

## Data Requirements

The pipeline expects `medicaid-provider-spending.csv` (~11 GB) in the project root. This is the CMS Medicaid Provider Utilization and Payments dataset.

Large data files are excluded from this repository via `.gitignore`:
- `medicaid-provider-spending.csv` (11 GB source data)
- `medicaid.duckdb` (30 GB analytical database)
- `claims.parquet` (2.9 GB)
- `reference_data/nppes/` (10 GB+ NPI registry)
- `output/findings/` (8 GB+, regenerated by the pipeline)

## Usage

```bash
# Place medicaid-provider-spending.csv in the project root, then:
python3 scripts/run_all.py
```

Individual milestones can also be run standalone:

```bash
python3 scripts/03_eda.py
python3 scripts/10_generate_charts.py
```

## Detection Methods

- **Statistical:** Z-scores, Benford's law, peer group deviation, Lorenz/Gini concentration
- **Machine Learning:** Isolation Forest, Random Forest, XGBoost anomaly scoring
- **Domain Rules:** Impossible daily volumes, billing after deactivation, circular billing networks
- **Cross-Reference:** LEIE exclusion list matching, NPPES registry validation, multi-method composite scoring
- **Longitudinal:** Panel regression, structural breaks, COVID-adjusted temporal analysis
